# Week 1 Review

## What Was Learned
- Software development processes organize project work.
- Agile and iterative models are useful when requirements may change.
- SOLID principles help keep software maintainable.
- Design patterns are useful only when they solve a real design problem.
- Git can be used as a learning and thinking history, not only as a code storage tool.

## Which Beliefs Survived Stress
- Iterative development is useful for learning projects.
- GitHub is appropriate for tracking course progress.
- Separating beliefs, decisions, and experiments improves clarity.

## Which Assumptions Failed
- Design patterns are always beneficial.
- Waterfall is always easier because it is linear.
- Notes are useful without synthesis.

## Next Actions
- Add more learning artifacts after each lecture.
- Record important project decisions.
- Create experiments when testing tools, patterns, or process models.
