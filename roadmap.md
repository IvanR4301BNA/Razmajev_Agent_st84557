# Roadmap (MVP)

## Stage 1 — Project Setup
- Create repository structure
- Set up frontend, backend, and database
- Define coding conventions and project rules
- Add initial documentation and AGENTS.md

## Stage 2 — Authentication
- User registration
- User login
- Session or token-based authentication
- Basic access protection for authenticated routes

## Stage 3 — Core Task Management
- Create task
- Edit task
- Delete task
- Assign task to user
- Set due date and priority
- Update task status: To Do, In Progress, Done

## Stage 4 — Dashboard and UI
- Task list view
- Status overview dashboard
- Filtering and sorting tasks
- Basic responsive interface

## Stage 5 — Quality and Testing
- Unit tests for services
- Integration tests for API endpoints
- Basic validation and error handling
- Final cleanup and refactoring

## MVP Scope
The MVP includes authentication, basic task CRUD operations, task assignment, status tracking, and a simple dashboard. The goal of the MVP is to deliver a working product with clear architecture and safe AI collaboration rules.

## Deferred to Future Releases
- Real-time notifications
- File attachments
- Team chat
- Advanced analytics
- Mobile application
- Third-party integrations
