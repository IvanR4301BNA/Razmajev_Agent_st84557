# Development Process Selection

## Decision Question
Which software development process should be used for learning and project work?

## Context Snapshot
The course covers software process models such as Waterfall, Iterative, Agile, Spiral, RAD, and V-Model. The project work is expected to evolve during learning.

## Alternatives Considered
- Waterfall
- Agile
- Spiral
- Iterative development

## Decision
Use an Agile / Iterative approach.

## Reasoning
Agile and iterative development allow regular feedback, small improvements, and easier adaptation when requirements are not fully clear.

## Expected Outcomes
- Faster learning.
- Better tracking of progress.
- Easier correction of mistakes.
- More visible development history in Git.

## Reconsideration Conditions
This decision should be reconsidered if the project requirements become fixed, simple, and unlikely to change.
