# Project Management Approach

This project follows a hybrid Agile approach that combines Scrum and Kanban. Scrum is used at the planning level to divide the work into clear stages and define the MVP, while Kanban is used during execution to manage the flow of work and adapt to changes.

The human developer is responsible for architecture, design decisions, task prioritization, code review, and final approval of all changes. AI agents are responsible for generating boilerplate code, documentation, tests, and assisting with refactoring and bug fixes. This division of labor allows the human developer to focus on high-level decisions while AI agents handle repetitive and time-consuming tasks.

The project is organized into clear stages that are defined in the Roadmap. Each stage has a specific goal and deliverables. The team uses GitHub Issues and Projects to track progress and manage the workflow.
