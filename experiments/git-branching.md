# Git Branching Experiment

## Actions Performed
- Created or planned a separate branch for testing changes.
- Added Markdown files to the repository.
- Prepared changes before merging them into the main branch.

## Measurable Results
- Changes can be reviewed before becoming part of the main repository.
- The main branch remains stable.
- Git history shows separate steps of work.

## Deviations From Expectation
No major deviations.

## Unexpected Signals
Branching is useful not only for code but also for testing changes in documentation and learning structure.
